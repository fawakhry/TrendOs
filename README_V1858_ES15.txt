TrendOS V1858 + EasyStore ES15 Ledger Fix

محتويات الباتش:
1) ملفات TrendOS الأساسية: index.html / app.js / styles.css / config.js.
2) ملف Google Apps Script: Google_Apps_Script_V1858_ES15_Ledger_Fix.gs.
3) مجلد EasyStore_ES15_V1858_Ledger_Fix_patch ويترفع على repo بتاع EasyStore.

مهم جدًا:
- مشكلة فاتورة العميل كانت لأن EasyStore ما زال يحمل V13 Batch32.
- بعد رفع EasyStore patch لازم فحص النظام يظهر ES15 V1858 Ledger Fix.
- لو ظهر V13 Batch32 يبقى المتصفح أو GitHub Pages ما زال ماسك نسخة قديمة.

الجديد في V1858/ES15:
- حسابات عملاء وموردين.
- إضافة مديونية للعميل بواسطة ضياء/رحمه/ريفان.
- تسجيل سداد العميل في أي وقت وتحديث الرصيد.
- تسجيل مستحق/دفعة للمورد في أي وقت وتحديث الرصيد.
- كشف حساب كامل لكل عميل أو مورد.
- صلاحيات: ضياء/رحمه/ريفان فقط للحسابات والمديونيات. وائل/جابر لا يعدلوا مديونيات.
- إصلاح أقوى لقائمة فاتورة العميل.


ES15.1 Ledger Placement Fix:
1) زر تعديل/مديونية أصبح في العملاء فقط وليس الأصناف.
2) زر حساب المورد يظهر في الموردين فقط.
3) تحسين تحميل بيانات العميل في فاتورة المبيعات.
4) ارفع فولدر EasyStore على repo EasyStore واضغط Ctrl+F5.
